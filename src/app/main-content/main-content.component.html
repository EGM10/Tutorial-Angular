¿Qué es un framework?

I. Concepto

Aunque un framework es una herramienta con muchos campos de aplicación, no solo en el desarrollo de páginas web, como introducción vamos a referirnos a ellas y decir que quien esté interesado en desarrollar páginas web tiene dos posibles caminos:

  1. Enfoque sencillo y rápido: destinado a quien desconociendo el código a implementar tiene las ideas claras de lo que debe aparecer en su web, en cuyo caso puede hacer uso de herramientas como Wix, Squarespace o WordPress, que no requieren conocimientos técnicos porque estas plataformas permiten crear sitios web reemplazando textos e imágenes predeterminadas mediante interfaces intuitivas. Por tanto, son ideales para personas o empresas que buscan soluciones rápidas sin preocuparse por el código. Otra posibilidad igual de rápida es recurrir a servicios en la nube, que proporcionan grandes compañías (AWS, Azure, etc.) porque:
    • Proveen servicios listos para usar, como alojamiento web, almacenamiento de datos o gestión de contenido.
    • Permiten ahorrar tiempo y recursos, pero suelen ser menos personalizables que un desarrollo a medida.
    Ambas opciones son ideales para quienes no necesitan funcionalidades avanzadas o no tienen conocimientos técnicos.
  2. Enfoque técnico y avanzado: implica aprender tecnologías fundamentales, tales como:
    • Frontend: HTML, CSS, JavaScript.
    • Backend: Node.js, PHP, Python, bases de datos, etc.
    • Herramientas avanzadas: librerías y frameworks.

    El conocimiento de estas tecnologías permite controlar todos los aspectos del diseño y funcionalidad de un sitio web, produciendo soluciones personalizadas, escalables y más optimizadas. Este es el camino recomendado para desarrolladores profesionales y quienes buscan libertad total para crear.

II. Campos de aplicación de un framework

Mostramos un resumen de los campos de aplicación de un framework y, dentro de ellos, las diferentes áreas de destino:

  1. Diseño y desarrollo de páginas web
    • Frontend frameworks: permiten manejar la lógica del lado del cliente, como Angular o Vue.js.
    • Backend frameworks: permiten manejar la lógica del servidor, como Django, Ruby on Rails o Express.js.
  2. Desarrollo de aplicaciones móviles
    • Frameworks multiplataforma: Flutter, React Native o Ionic.
    • Frameworks específicos: SwiftUI y Jetpack Compose.
  3. Desarrollo de aplicaciones de escritorio
    • Frameworks como Electron o Qt.
  4. Inteligencia artificial y machine learning
    • TensorFlow Extended (TFX) y Apache Spark MLlib.
  5. Desarrollo de videojuegos
    • Unity, Unreal Engine y Godot.
  6. Desarrollo de sistemas embebidos
    • MBED OS y Zephyr.
  7. Automatización y pruebas
    • Selenium y Robot Framework.
  8. Ciencia de datos y visualización
    • KNIME y Apache Superset.
  9. Desarrollo de APIs y microservicios
    • FastAPI, Spring Boot y Micronaut.
  10. Automatización de infraestructura y DevOps
    • Ansible, Terraform y Chef.

III. Estructuras de un framework

"Estructura de trabajo" en un framework significa que existen normas en las siguientes cuestiones:

  1. Organización del proyecto: El framework dicta cómo se deben estructurar los directorios, los nombres de los archivos y dónde colocar cada componente.
  2. Patrones de diseño predefinidos: MVC, MVVM, REST.
  3. Convenciones en lugar de configuraciones: Estandarizaciones como las de Spring Boot.
  4. Flujo de trabajo definido: Definición clara de cómo realizar tareas específicas.

IV. Principales particularidades de un framework

  • Estructura predefinida
  • Inversión de control (IoC)
  • Funcionalidades integradas
  • Modularidad
  • Estándares y convenciones
  • Automatización de tareas comunes
  • Integración con herramientas externas
  • Comunidad y ecosistema
  • Seguridad
  • Escalabilidad

V. Diferencias entre framework y librería

Tanto las librerías como los frameworks son herramientas que proporcionan estructuras y funcionalidades prediseñadas para simplificar y acelerar el desarrollo de una aplicación cualquiera (desde el diseño web hasta el desarrollo de sistemas de inteligencia artificial, videojuegos o automatización de procesos), pero tienen diferencias importantes:

  • Librerías:

    Son bloques de código reutilizables que cumplen una determinada función; es decir, resuelven asuntos concretos. Por ejemplo, una librería podría gestionar un carrito de la compra en una tienda online. Dentro de ese código puede incluirse una función para verificar si hay stock del producto, otra para concretar la compra, e incluso otra para actualizar el stock en el almacén. Todo ello podría empaquetarse en una biblioteca. Otros bloques de código que resuelven diferentes asuntos, como mostrar un calendario o un gráfico por pantalla, pertenecerán a otras librerías.

    Una librería muy conocida es React, desarrollada por Facebook, usada específicamente para construir interfaces de usuario (UIs) basadas en componentes reutilizables.

  • Frameworks:

    Los frameworks también pueden usar librerías (de hecho, casi todos los usan), pero son aplicaciones más completas porque incluyen muchos tipos de librerías, todas compatibles entre sí, algo que no siempre ocurre con librerías independientes. Sin embargo, un framework es mucho más que un conjunto de librerías compatibles; es un entorno completo que proporciona una estructura y herramientas para desarrollar aplicaciones de manera más eficiente y estandarizada. Los frameworks se utilizan en muchos campos de aplicación y establecen una estructura y un flujo de trabajo que el desarrollador debe seguir, delegando el control al framework para la ejecución del código.

    En otras palabras, en una librería, el desarrollador tiene el control, mientras que en un framework, es este el que controla cómo interactúa el código del desarrollador con el resto del sistema.

Las librerías generalmente no tienen una estructura definida como los frameworks. Esto se debe a que las librerías no imponen un flujo de trabajo ni una organización específica para el código del desarrollador. En lugar de eso, ofrecen funciones, clases o utilidades específicas que el desarrollador puede usar libremente en su proyecto según sus necesidades.

I. Introducción

ANGULAR: Angular es un framework de desarrollo web basado en TypeScript, mantenido por Google. Se utiliza para crear aplicaciones web dinámicas y modernas de una sola página.

II. Instalación

Tiene los siguientes prerrequisitos:

  • Node.js - v^18.19.1 o más reciente
  • Editor de texto: se recomienda Visual Studio Code (VSC)
  • Terminal: necesario para ejecutar comandos CLI de Angular

Pasos para instalar Angular:

  1. Instalar Angular CLI: 
    npm install -g @angular/cli
  2. Crear una carpeta para el proyecto: 
    Nos situamos en la ruta deseada y ejecutamos: 
        md nombre-de-carpeta
  3. Crear un nuevo proyecto Angular: 
    Nos situamos en la carpeta deseada y ejecutamos:
        ng new nombre-proyecto
  4. Navegar al proyecto y ejecutar la aplicación: 
    nos situamos en la carpeta que contiene el proyecto y ejecutamos:
ng serve

Al ejecutar ng serve, Angular compila y sirve la aplicación en http://localhost:4200.

III. Tutorial Sorpresa

Cuando usamos el comando ng serve del paso anterior, Angular CLI genera automáticamente un completo tutorial interactivo que se muestra en la página de inicio de la aplicación. Este tutorial incluye botones y enlaces para explorar documentación oficial y realizar pruebas con códigos interactivos.

IV. Estructura de un Proyecto Angular

La estructura básica de un proyecto Angular es la siguiente:

  • nombre-proyecto/
    • e2e/ # Pruebas end-to-end (opcional)
    • node_modules/ # Dependencias instaladas
    • src/
      • app/
        • app.component.css
        • app.component.html
        • app.component.spec.ts
        • app.component.ts
        • app.module.ts
      • assets/
      • environments/
        • environment.ts
        • environment.prod.ts
      • index.html
      • main.ts
      • styles.css
      • test.ts
    • angular.json
    • package.json
    • tsconfig.json
    • karma.conf.js
